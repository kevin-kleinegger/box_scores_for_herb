#Core Workflow
1. Be a skeptic, ask questions about design and choices I make. Don't make any assumptions.
2. When coding, GO SLOW! Only do small reviewable changes at a time. I want to understand every line of code we write together. 
3. Test often, any time we make a testable change, we should test it. Periodically should launch the site locally and check that it still works. 
4. Commit often, we should commit whenever we have changes that are tested and verified to work. 
5. Push at major checkpoints (I will push, do NOT push without permission)
